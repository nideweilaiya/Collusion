# 🗺️ Collusion Roadmap（共谋路线图）

这份路线图记录了目前已经讨论过、尚未实现但计划在未来版本中完成的功能和改进。每一项都源于真实测试中的发现和社区场景的预判。

---

## 近期目标 (v0.4.0 - v0.5.0)

### HTML 可视化报告

- 当前方案产出的是 Markdown 文本
- 计划使用 Jinja2 模板引擎生成单文件 HTML，包含雷达图（方案多维对比）、架构分层图（Mermaid.js）、风险标注卡片和可离线查看的完整内容
- 预计增加 token 消耗约 20-30%，但提供离线可用性和更高的信息密度

### 可执行 JSON 蓝图

- 为 Top 1 方案自动生成结构化任务清单
- 包含任务 ID、名称、描述、依赖关系、预计耗时、优先级
- 可直接被 Superpowers 的 writing-plans 或 Claude Code 的 Plan 模式解析执行
- 用户无需手动整理任务清单

### 用户反馈回路

- 在 HTML 报告的每个步骤中嵌入"修改建议"输入区域
- 用户提交修改意见后，新增的 `collusion_refine` 工具触发三个对象代言人独立审查
- 每个 Agent 返回认可、有隐患、高创新性三种反馈之一，附详细理由
- 用户根据多视角反馈矩阵决定是否采纳修改，而非重新生成整个方案
- 反馈模式成本极低（3 个 Agent 各输出约 100-200 字），不与现有编排流程冲突

### 成本控制与缓存优化

- 固化为全局前缀，实现字节级稳定
- 修改阶段输入瘦身，只传入相关环节片段而非全方案
- 目标：单任务成本从当前约 ¥0.15 降至 ¥0.05-0.10

## 中期目标 (v0.6.0 - v0.8.0)

### 增量增强模式（基于现有设计）

- 当前编排流程仅支持从零开始生成方案，不支持接收已有半成品方案进行增强
- 计划新增 `collusion_enhance` 工具：用户传入一份已有方案，引擎调度多个对象代言人进行多视角审查、补充不足、优化结构
- 区别于 `collusion_refine`（用户反馈回路，针对具体环节的修改建议），`collusion_enhance` 是宏观层面的方案增强
- 适用场景：用户已有初步设计，希望 AI 从安全、性能、成本等角度全面审查和完善
- 技术上需要新增一个"方案注入"入口，跳过 Phase 1-3，直接从 Phase 4（交叉审查）开始

### 动态 Agent 调度

- 当前 Agent 数量和角色是硬编码的，未来根据任务类型和复杂度自动选择需要几个 Agent、分别扮演什么角色
- 简单任务用 2 个 Agent，复杂任务扩展到 5 个
- 提供配置项 `num_agents` 和 `agent_roles`，让用户自定义团队

### Agent 角色扩展

- 当前三个角色：安全专家、性能架构师、UX/产品专家
- 计划新增：成本优化师、前端专家、数据工程师、DevOps 顾问
- 允许用户通过配置文件注册自定义角色和提示词

### 多平台接入优化

- 当前已支持 stdio 传输（Trae Solo 已验证），计划增加 HTTP + SSE 传输，支持远程调用和云端部署
- 适配 Reasonix Hook 集成，利用其极致缓存降低重复任务成本
- 完善 MCP 市场提交（MCP Market、mcpservers.org）

## 远期愿景 (v1.0.0+)

### 与执行工具的深度集成

- 与 Superpowers 的 writing-plans 和 executing-plans 无缝衔接，提供一键传输方案到执行的能力
- 与 OpenSpec 的 Spec-driven development 流程集成，让方案设计成为 Spec 的前置步骤
- 与 gstack 等团队协作工具组合，覆盖"设计 → 规范 → 执行 → 团队协作 → QA → 发布"的完整链路

### 自进化方案库

- 收集用户采纳的方案和实际落地反馈
- 建立方案-场景匹配模型，为类似任务推荐历史最优方案
- 积累"可行性收束"的成功案例库，提升收束建议的精准度

### 多语言方案输出

- 当前系统提示词和输出均为英文，计划支持中文方案输出
- 中英文版本通过配置文件切换，降低国内用户的使用门槛

## 📋 对社区的承诺

这份路线图是公开的。每一项功能都来自真实需求或社区反馈，不接受凭空想象的 PR。如果你对某个方向有建议或想参与开发，请在 Issue 中提出，或在 Discussion 中发起讨论。

当前版本 (v0.3.1) 已经实现的核心能力：

- 多对象代言并行提案
- 环节共识与缺失补全
- 交叉审查与可行性强制收束
- 多维度投票评分，输出 Top 3 方案
- 与 Superpowers 的盲评对比中 6:1 胜出（开源博客平台任务）

欢迎 Watch、Star、Fork，一起让"共谋"成为 AI 编码工作流中最可靠的方案设计引擎。
