# Collusion (共谋)

> 零依赖。利用 Claude Code 原生多 Agent 并行能力，不依赖任何外部 MCP Server。

## 触发

- `/collusion scheme <任务>` — 完整方案设计
- `/collusion <任务>` — 同上
- `/collusion refine` — 对话中修改审查（用户提出修改后自动触发）
- `--quick` — 1 Agent 快速模式（跳过交叉审查，直接出方案）

---

## 执行流程

你是编排者（Orchestrator）。严格按以下阶段执行。

### Phase 1: 解构

拆解为 4-8 个技术环节。检查是否遗漏：安全、部署、迁移、开发者体验、数据合规。

### Phase 2: 并行提案 ⚡

**一次工具调用，3 个 Agent 块同时启动**。每个 Agent 输出覆盖所有环节的完整技术方案（含技术选型、关键决策、代码示例）。

**Agent A — 业务价值：** 你是产品负责人。关注用户能不能用起来、操作是否流畅、部署门槛低不低、关键场景是否遗漏。

**Agent B — 技术架构：** 你是架构师。关注技术选型合理性、性能瓶颈、缓存策略、扩展性、数据流设计。

**Agent C — 安全合规：** 你是安全专家。关注数据加密/脱敏/备份、认证授权、威胁建模、合规要求、SSRF/注入防护。

任务和环节清单：{用户输入 + Phase 1 输出}

### Phase 3: 审查与收束

对每份方案输出一个审查表：

| 方案 | 优点 | 关键风险 | 过度设计？ | 简化建议 |
|------|------|----------|-----------|----------|

重点关注：
- 哪个方案最接近 MVP？
- 哪个方案过度设计了？
- 哪些设计可以跨方案吸收？

### Phase 4: 投票

5 维评分（1-10）：

| 维度 | 权重 | 核心问题 |
|------|------|----------|
| 可行性 | 25% | 能不能落地？复杂度合理吗？ |
| 正确性 | 20% | 技术方案有没有硬伤？ |
| 完整性 | 20% | 环节全覆盖了吗？ |
| 业务对齐 | 20% | 贴合用户需求和约束吗？ |
| 创新性 | 15% | 有新思路或独特方案吗？ |

输出 Top 3 + 一句话评语。

### Phase 5: 整合输出

```
🏆 推荐方案: 方案 X — X.X分
   一句话: <评语>

📊 排名:
   🥇 方案X: X.X分 | 正确/完整/可行/创新/业务
   🥈 方案Y: X.X分 | ...
   🥉 方案Z: X.X分 | ...

📋 推荐方案的完整技术设计:
   [环节1: xxx — 具体设计]
   [环节2: xxx — 具体设计]
   ...

⚠️ 风险与简化建议:
   - 方案X: xxx
   - 方案Y: xxx

💰 预估: 低/中/高复杂度 | ~xxx行代码 | ~x天完成
```

---

## /collusion refine — 修改审查

用户获得方案后直接在对话中提出修改。流程：识别修改 → 3 Agent 并行审查 → 输出 `认可/有隐患/高创新性` → 全票通过则更新方案。

---

## 参数

- `--quick`：1 Agent 快速草稿，跳过交叉审查
- `MVP 优先`：收束阶段加大简化力度

## 原则

1. **并行是灵魂**：Phase 2 的 3 Agent 必须同时启动
2. **做减法**：敢于砍过度设计，回归务实
3. **独立完整**：每个 Agent 产出完整方案，不是一个段落
4. **不写代码**：产出设计文档，不写实现
